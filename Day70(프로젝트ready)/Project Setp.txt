PM 프로그램계획서 만드는사람
보통 계발표준정의서를 제공하고 따라서 만들게됨
초반에 회사에선 샘플코드도 만들어서 제공하기도함

Project Setp

설계가 잘되야 프로젝트가 잘만들어지고 보통 설계하는데 시간의 절반을 사용하기때문에
천천히 세세하게 만들기


1. 착수단계
- 산출물 : WBS, 수행계획서, 형상관리계획서, 품질보증계획서
2. 분석단계
- 산출물 : 인터뷰, 요구사항정의, 현행시스템분석
3. 설계단계
- 산출물 : 화면설계, 데이터베이스설계, 프로그램설계, 인터페이스설계
4. 구현단계
- 산출물 : 코딩, 단위테스트
5. 테스트단계
- 산출물 : 통합테스트, 시스템테스트
6. 이행단계
- 산출물 : 전환, 인수시험, 지침서, 유지보수
- 위단계(전체 프로세스)
- 일정, 범위, 의사소통, 품질, 위험/이슈, 문서관리, 형상관리, 자원관리, 교육훈련, 보안, 조달
7. 오픈단계
- 오픈공유, 개발, 통합테스트, 인프라, 시스템 전환, 데이터 전환, 이슈
8. 인수단계
- 설계산출물, 인수인계, 유지보수
