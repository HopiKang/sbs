java class 첫시작

java란?

java는 모든 환경에 호환된다
언어순위 3위권안에 계속 들어가있었다
소규모보다 대규모 플렛폼에 어울린다
전자정부 표준프레임워크에서 java개활환경을 설명해주고있다
jvm(java프로그램이 돌아가기위한 환경)  jre(jvm을 포함한다)
java는 11버전을 사용한다


환경변수 설정
시스템 속성에 > 고급 > 환경변수 > 시스템 변수 새로만들기
>1. JAVA_HOME - 사용할 java 경로 
>2. 위 사용자변수 만들기 > JAVAPATH - .,%JAVA_HOME%\lib\tools.jar;
>3. 위 사용자변수에 path 편집 > 새로만들기 > %JAVA_HOME%\bin\

디버깅이란
문제를 고쳐나가는 과정

빌드란
소스코드를 비롯한 모든 프로그램을 모아서 실행할수있는 파일묶음을
만드는 과정

jar와 war

배포
빌드를 마친 실행가능한 파일을 사용자가 접근가능한 환경에 배치하는것
(서버에 업로드하는것)

리팩토링이란
코드를 추가하거나 수정해야할경우 쉽게할수있게(역할을 잘 수행할수있게) 
코드를 간결하고 효율적으로 만드는것

컴파일오류
소스코드의 오타나 문법적 결함일때 나타나는 오류
컴파일할때 위에서 아래로 왼쪽에서 오른쪽으로 읽을때 오타를 나타내준다

런타임오류
컴파일이없는 인터프린터언어에서 발생하는오류
발견하지 못하고 프로그램이 배포될수있다
대표적으로는 null오류나 /0오류 무한루프오류가있다

버그란
프로그램이 의도와 다르게 작동하는것

메모리 
가비지컬렉션이란
사용한 메모리가 필요없게되면 할당된 메모리영역을 해제하는 기능

메모리누수
메모리공간에 불필요한 데이터가 쌓여서 남아있는것

가비지컬렉션
메모리위에 필요없는 데이터를 프로그램 스스로 치워주는것
프로그램이 작동하는 도중에 가동되면 프로그램이 멈추기도한다
(메모리누수가 생길수있다)

패키지매니저
수많은 외부라이브러리를 가져와서 사용하는데 필요한 라이브러리를
일일이 기억할수없으니 그걸모아서 관리할수있게해준다
(소스코드를 배포하는것)
프로그램(언어), 운영제제에서 사용하는것으로 나뉜다
-운영체제 리눅스(apt-get,yum,pacman)

라이브러리
소스코드를 작성하기위해서 사용되는 코드의 묶음

환경변수란
프로그램이 컴퓨터에서 작동할때 환경과 관련된 변수이다
운영체제의 응용프로그램을 쉽게쓰기위해서 경로를 지정해주는것이다
(변수지정을 미리해두는것)
(intellij사용할때 따로 환경변수를 지정하지않는 이유는 프로그램안에서
따로 경로지정을 해주기때문에 다른 프로그램을 사용할경우 충동을
일으킬수있기때문)

jdk(컴파일, 디버그, 개발도구)
> jre(자바라이브러리)
> jvm

sdk란
